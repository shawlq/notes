# 4. 自动驾驶特定方法

## 4.1 传感器数据处理

### 4.1.1 LiDAR 点云处理

- **名称**：点云坐标系与标定处理
  - **简介**：将 LiDAR 点从传感器坐标系转换到车体、世界或 BEV 坐标系，依赖外参标定和时间戳补偿。
  - **适用场景**：多传感器融合、3D 检测、轨迹回放、离线数据挖掘。
  - **优势或局限**：几何解释性强；标定漂移、车辆运动补偿和点云稀疏性会直接影响训练标签与推理质量。
  - **参考资料链接**：https://www.open3d.org/docs/latest/ 、https://www.cvlibs.net/datasets/kitti/

- **名称**：体素化与柱状编码
  - **简介**：将无序点云划分为 voxel 或 pillar，再用稀疏卷积、PointNet 或 Transformer 编码。
  - **适用场景**：CenterPoint、SECOND、PointPillars、BEV 检测模型训练与部署。
  - **优势或局限**：利于 GPU 批处理和 BEV 特征构建；分辨率、范围裁剪和空体素策略会影响精度与延迟。
  - **参考资料链接**：https://arxiv.org/abs/1812.05784 、https://arxiv.org/abs/2006.11275

- **名称**：点云质量检查
  - **简介**：统计点数、反射强度、空帧、时间戳、异常距离和 sensor dropout，形成数据准入规则。
  - **适用场景**：自动驾驶数据闭环、训练集构建、仿真与实车日志回灌。
  - **优势或局限**：能提前过滤硬件异常和标注风险；需要结合车型、安装位和天气分桶设置阈值。
  - **参考资料链接**：https://www.nuscenes.org/ 、https://waymo.com/open/

### 4.1.2 Camera 图像处理

- **名称**：相机内外参与去畸变
  - **简介**：基于内参、畸变参数和外参对图像进行几何校正，并与 LiDAR/Radar 对齐。
  - **适用场景**：多视角感知、BEVFormer/BEVFusion、投影标注检查。
  - **优势或局限**：是跨模态融合的基础；标定误差在远距离目标上会被放大。
  - **参考资料链接**：https://docs.opencv.org/4.x/d9/d0c/group__calib3d.html

- **名称**：多相机环视预处理
  - **简介**：对前后左右多路相机进行 resize、normalize、padding、同步采样和相机顺序标准化。
  - **适用场景**：多视角 3D 检测、车道线、占用网络、端到端驾驶模型。
  - **优势或局限**：统一输入规范便于批训练；大分辨率和多路输入会显著增加显存与数据加载压力。
  - **参考资料链接**：https://arxiv.org/abs/2203.17270 、https://arxiv.org/abs/2205.13542

- **名称**：恶劣天气与曝光增强
  - **简介**：针对夜间、逆光、雨雾、脏污和 motion blur 做数据增强或图像恢复。
  - **适用场景**：长尾场景训练、鲁棒性 benchmark、自动驾驶仿真域迁移。
  - **优势或局限**：能提升覆盖率；过强增强可能引入不真实纹理，需用场景分桶评估。
  - **参考资料链接**：https://albumentations.ai/docs/ 、https://docs.nvidia.com/drive/

### 4.1.3 Radar 数据处理

- **名称**：Radar 点迹过滤与速度补偿
  - **简介**：处理雷达点迹的位置、径向速度、RCS、噪声点和 ego-motion 补偿。
  - **适用场景**：高速目标感知、雨雾天气冗余感知、低成本量产感知栈。
  - **优势或局限**：速度观测稳定、天气鲁棒；角分辨率和语义信息弱，需与 Camera/LiDAR 融合。
  - **参考资料链接**：https://www.nuscenes.org/nuscenes 、https://arxiv.org/abs/2005.07431

- **名称**：Radar-Camera/LiDAR 融合
  - **简介**：将 Radar 点迹投影到图像或 BEV，与视觉/点云特征进行前融合、中融合或后融合。
  - **适用场景**：远距离车辆检测、恶劣天气感知、速度辅助跟踪。
  - **优势或局限**：补充速度和穿透性信息；同步、标定和稀疏匹配难度较高。
  - **参考资料链接**：https://arxiv.org/abs/2203.12602 、https://pmc.ncbi.nlm.nih.gov/articles/PMC12074113/

## 4.2 感知方法

### 4.2.1 目标检测

- **名称**：2D/3D 目标检测
  - **简介**：识别车辆、行人、骑行者、交通锥等目标，并输出 2D 框、3D 框或 BEV 框。
  - **适用场景**：自动驾驶感知主干、数据挖掘、场景重建、仿真回放。
  - **优势或局限**：任务定义清晰、指标成熟；长尾类别、小目标、遮挡和远距离仍是难点。
  - **参考资料链接**：https://arxiv.org/abs/2006.11275 、https://arxiv.org/abs/2203.17270

- **名称**：BEV 感知
  - **简介**：将多相机或多传感器特征统一到鸟瞰图空间，服务检测、分割、占用和规划。
  - **适用场景**：多视角相机 3D 检测、融合感知、端到端驾驶中间表征。
  - **优势或局限**：空间表达适合驾驶决策；视角转换和深度估计会引入误差。
  - **参考资料链接**：https://arxiv.org/abs/2203.17270 、https://arxiv.org/abs/2205.13542

- **名称**：Occupancy Network
  - **简介**：预测 3D 空间体素或 BEV 网格的占用、语义和可通行状态。
  - **适用场景**：复杂静动态障碍物、无规则目标、规划安全边界。
  - **优势或局限**：比固定类别框更通用；标注成本、分辨率和实时性压力更高。
  - **参考资料链接**：https://arxiv.org/abs/2304.14365 、https://arxiv.org/abs/2302.07817

### 4.2.2 目标跟踪

- **名称**：Tracking-by-Detection
  - **简介**：先逐帧检测目标，再用 Kalman Filter、匈牙利匹配、IoU/外观/速度特征关联轨迹。
  - **适用场景**：量产感知栈、benchmark 评测、可解释跟踪系统。
  - **优势或局限**：工程简单、可调试；检测漏检和遮挡会造成 ID switch。
  - **参考资料链接**：https://github.com/ifzhang/ByteTrack 、https://arxiv.org/abs/2206.09467

- **名称**：联合检测跟踪
  - **简介**：模型同时输出检测框和跨帧关联特征，减少独立模块误差累积。
  - **适用场景**：端到端感知、长时序 BEV 模型、多任务训练。
  - **优势或局限**：可共享时序特征；训练标签、显存和调参复杂度更高。
  - **参考资料链接**：https://arxiv.org/abs/2308.05713 、https://link.springer.com/article/10.1007/s11633-025-1558-0

### 4.2.3 感知融合

- **名称**：Early/Middle/Late Fusion
  - **简介**：分别在原始数据、特征层或决策层融合 Camera、LiDAR、Radar 信息。
  - **适用场景**：多传感器检测、恶劣天气冗余、不同成本车型感知方案。
  - **优势或局限**：融合层级越早信息越完整但对齐难度越高；后融合稳定但表达能力有限。
  - **参考资料链接**：https://pmc.ncbi.nlm.nih.gov/articles/PMC12074113/ 、https://openreview.net/forum?id=h8qyde9kEA

- **名称**：Transformer 跨模态融合
  - **简介**：使用 cross-attention、query 或 BEV token 汇聚不同视角和传感器特征。
  - **适用场景**：BEVFusion、TransFusion、多相机多雷达统一感知。
  - **优势或局限**：建模能力强；计算量、延迟和部署优化难度较高。
  - **参考资料链接**：https://arxiv.org/abs/2203.11496 、https://arxiv.org/abs/2205.13542

- **名称**：时序融合
  - **简介**：融合多帧 BEV、点云 sweep 或图像特征，利用历史信息提升稳定性。
  - **适用场景**：遮挡恢复、速度估计、低帧率传感器补偿、占用预测。
  - **优势或局限**：能提升平滑性和召回；需要处理 ego-motion、缓存和延迟预算。
  - **参考资料链接**：https://arxiv.org/abs/2206.10092 、https://arxiv.org/abs/2308.05713

## 4.3 训练策略

### 4.3.1 端到端训练

- **名称**：端到端感知-预测-规划
  - **简介**：使用统一网络从传感器输入学习感知、预测、规划或控制输出，减少模块级误差传递。
  - **适用场景**：研究型自动驾驶系统、闭环仿真、可微规划探索。
  - **优势或局限**：可联合优化最终驾驶目标；可解释性、数据规模、安全验证和部署风险更高。
  - **参考资料链接**：https://arxiv.org/abs/2212.10156 、https://link.springer.com/article/10.1007/s11633-025-1558-0

- **名称**：World Model / Foundation Model Driving
  - **简介**：通过大规模视频、BEV 或多模态 token 建模动态世界，用于预测、规划和仿真。
  - **适用场景**：闭环策略学习、场景生成、长尾数据增强、基础模型预训练。
  - **优势或局限**：有潜力提升泛化；训练成本高，真实闭环安全评估仍困难。
  - **参考资料链接**：https://arxiv.org/abs/2311.18608 、https://arxiv.org/abs/2405.05949

### 4.3.2 模块化训练

- **名称**：感知模块独立训练
  - **简介**：按检测、分割、跟踪、预测等任务分别训练模型，并通过接口集成到系统。
  - **适用场景**：量产工程、可解释评测、按团队分工迭代的感知栈。
  - **优势或局限**：边界清晰、便于验证和回归；模块误差会累积，接口设计影响系统上限。
  - **参考资料链接**：https://tier4.github.io/autoware-ml/ 、https://github.com/open-mmlab/mmdetection3d

- **名称**：多任务联合训练
  - **简介**：共享 backbone，同时训练检测、分割、深度、占用、车道线等任务头。
  - **适用场景**：算力受限部署、多任务感知平台、统一 BEV 特征。
  - **优势或局限**：提升特征复用和部署效率；任务 loss 平衡和负迁移需要重点调参。
  - **参考资料链接**：https://arxiv.org/abs/2205.13542 、https://arxiv.org/abs/2302.07817

### 4.3.3 自监督 / 半监督训练

- **名称**：自监督表征学习
  - **简介**：利用掩码建模、对比学习、时序一致性等目标，从未标注驾驶数据中学习特征。
  - **适用场景**：海量未标注车队数据、低标注预算、新城市域适配。
  - **优势或局限**：降低标注依赖；预训练任务与下游任务不匹配时收益有限。
  - **参考资料链接**：https://arxiv.org/abs/2111.06377 、https://arxiv.org/abs/2209.07636

- **名称**：半监督与伪标签训练
  - **简介**：用 teacher model 生成伪标签，与人工标注数据混合训练，并通过置信度过滤控制噪声。
  - **适用场景**：长尾目标扩充、跨地域迁移、低频场景数据利用。
  - **优势或局限**：可快速放大训练集；伪标签偏差会被学生模型继承，需要持续评估。
  - **参考资料链接**：https://arxiv.org/abs/2006.06882 、https://arxiv.org/abs/2303.17905
