# 2. 数据处理与增强

## 2.1 数据标注工具

### 2.1.1 LabelStudio

- **名称**：Label Studio
- **简介**：开源数据标注平台，支持图像、视频、文本、音频、多模态任务和自定义标注模板，可连接 S3、GCS、Azure Blob、PostgreSQL 等存储。
- **适用场景**：自动驾驶图像框/分割、多传感器样本审核、corner case 数据回流、人机协同标注质检。
- **优势或局限**：优势是任务类型覆盖广、模板灵活、便于快速搭建标注工作流；局限是大规模视频/点云标注和复杂质检流程通常需要二次开发或企业版能力。
- **参考资料链接**：[Label Studio Docs](https://labelstud.io/guide/)、[Cloud and external storage](https://labelstud.io/guide/storage)、[Import data](https://labelstud.io/guide/tasks)

### 2.1.2 CVAT

- **名称**：Computer Vision Annotation Tool（CVAT）
- **简介**：OpenCV 生态的开源计算机视觉标注工具，支持图像、视频、目标检测、分割、关键点、跟踪和自动标注插件。
- **适用场景**：车载相机数据标注、视频目标跟踪标注、检测/分割数据集构建、自动驾驶视觉样本返修。
- **优势或局限**：优势是视觉标注能力成熟、视频跟踪工作流强；局限是原生多传感器融合标注能力有限，点云/时序同步场景需要扩展。
- **参考资料链接**：[CVAT Docs](https://docs.cvat.ai/docs/)、[CVAT GitHub](https://github.com/cvat-ai/cvat)、[OpenCV CVAT](https://www.cvat.ai/)

## 2.2 数据预处理与增强

### 2.2.1 数据清洗与质量检查

- **名称**：数据清洗与质量检查
- **简介**：围绕重复样本、坏帧、传感器缺失、标注冲突、时间戳异常、类别长尾和分布漂移建立可自动执行的数据质量规则。
- **适用场景**：训练集入库前检查、主动学习样本筛选、回归测试数据集冻结、模型异常 case 归因。
- **优势或局限**：优势是能显著降低训练噪声、提升可复现实验可信度；局限是规则需要结合传感器、标注规范和业务场景持续维护。
- **参考资料链接**：[TensorFlow Data Validation](https://www.tensorflow.org/tfx/data_validation/get_started)、[Great Expectations](https://docs.greatexpectations.io/)、[Cleanlab](https://docs.cleanlab.ai/)

### 2.2.2 数据增强方法（旋转、裁剪、噪声）

- **名称**：数据增强方法
- **简介**：通过图像几何变换、颜色扰动、噪声注入、点云旋转/缩放/采样、雷达噪声模拟等方式扩充训练分布。
- **适用场景**：感知检测/分割训练、稀有天气和姿态覆盖、鲁棒性测试、长尾类别增强。
- **优势或局限**：优势是成本低、能缓解过拟合；局限是不符合物理约束的增强会破坏传感器几何一致性，自动驾驶多模态增强必须同步更新标定和标签。
- **参考资料链接**：[Albumentations Docs](https://albumentations.ai/docs/)、[TorchVision Transforms](https://pytorch.org/vision/stable/transforms.html)、[OpenPCDet Data Augmentation](https://github.com/open-mmlab/OpenPCDet)

### 2.2.3 传感器数据对齐与同步

- **名称**：传感器数据对齐与同步
- **简介**：对 Camera、LiDAR、Radar、IMU、GNSS 数据做时间同步、坐标变换、外参校准和 motion compensation，保证跨模态样本可融合。
- **适用场景**：多传感器融合训练、BEV 构建、点云投影到图像、时序跟踪和闭环仿真回放。
- **优势或局限**：优势是融合模型可信度的基础；局限是硬件时钟漂移、rolling shutter、车辆运动和标定老化都会引入系统性误差。
- **参考资料链接**：[ROS message_filters](https://docs.ros.org/en/rolling/p/message_filters/)、[OpenCV Camera Calibration](https://docs.opencv.org/4.x/dc/dbb/tutorial_py_calibration.html)、[nuScenes devkit](https://github.com/nutonomy/nuscenes-devkit)

## 2.3 数据存储与可视化

### 2.3.1 数据库与格式（Parquet、HDF5）

- **名称**：Parquet / HDF5
- **简介**：Parquet 是列式存储格式，适合大规模表格和特征数据分析；HDF5 是层次化二进制格式，适合数组、张量和传感器记录。
- **适用场景**：训练样本索引、特征缓存、标注元数据、离线评测结果、点云/图像派生数据存储。
- **优势或局限**：Parquet 查询和压缩效率高但不适合频繁小文件随机写；HDF5 对数组友好但并发写和云原生对象存储体验不如 Zarr/Parquet。
- **参考资料链接**：[Apache Parquet](https://parquet.apache.org/docs/)、[PyArrow Parquet](https://arrow.apache.org/docs/python/parquet.html)、[HDF5 Docs](https://docs.hdfgroup.org/hdf5/latest/)

### 2.3.2 可视化工具（Matplotlib、Open3D、Plotly）

- **名称**：Matplotlib / Open3D / Plotly
- **简介**：Matplotlib 用于离线图表和训练曲线，Open3D 用于点云/3D 几何处理与可视化，Plotly 用于交互式图表和 Web 报告。
- **适用场景**：训练监控报表、点云和 3D bbox 可视化、误检漏检分析、数据集分布审查。
- **优势或局限**：优势是覆盖 2D、3D、交互式分析；局限是大规模点云和长视频可视化需要采样、缓存或专用 WebGL 工具。
- **参考资料链接**：[Matplotlib Docs](https://matplotlib.org/stable/)、[Open3D Docs](https://www.open3d.org/docs/)、[Plotly Python](https://plotly.com/python/)

### 2.3.3 数据版本管理（DVC）

- **名称**：DVC
- **简介**：面向机器学习的数据版本管理工具，使用 Git 追踪元数据，将大文件存储在本地、S3、GCS、Azure、SSH 等 remote。
- **适用场景**：数据集快照、训练流水线依赖追踪、模型和数据共同复现、多人协作样本集管理。
- **优势或局限**：优势是与 Git 工作流结合紧密、适合可复现实验；局限是超大规模对象存储和复杂权限治理需要配套数据湖/特征平台。
- **参考资料链接**：[DVC Docs](https://dvc.org/doc)、[Data management](https://dvc.org/doc/user-guide/data-management)、[Pipelines](https://dvc.org/doc/user-guide/pipelines)
