## 1) 模型总体结构（端到端多任务）

`SparseDrive` 主干是：

- 图像编码：`ResNet50 + FPN`
- 稀疏感知头：`SparseDriveHead`
  - `det_head`：3D检测/跟踪（Sparse4DHead）
  - `map_head`：在线地图矢量化（Sparse4DHead）
  - `motion_plan_head`：运动预测 + 规划（并行）
- 可选辅助深度分支：`DenseDepthNet`（仅训练监督）

核心前向路径就是“提特征 -> 多任务head -> loss/post_process”：

```98:120:SparseDrive/projects/mmdet3d_plugin/models/sparsedrive.py
def forward_train(self, img, **data):
    feature_maps, depths = self.extract_feat(img, True, data)
    model_outs = self.head(feature_maps, data)
    output = self.head.loss(model_outs, data)
    if depths is not None and "gt_depth" in data:
        output["loss_dense_depth"] = self.depth_branch.loss(
            depths, data["gt_depth"]
        )
    return output
```

## 4) 架构细节（稀疏查询与时序）

### 4.1 det/map：对称 Sparse4DHead
两个头共享范式：

- `InstanceBank` 提供稀疏锚点查询
- `operation_order` 反复执行 `temp_gnn/gnn/deformable/ffn/refine`
- `refine` 同时产出分类与回归
- `det` 用 900 anchors，`map` 用 100 anchors
- decoder层数 6（前1层单帧，后续带时序）

### 4.2 motion_plan：并行运动预测+规划
`MotionPlanningHead` 读取 det/map 输出后：

- 从 det/map 挑 top-k（默认 det 50, map 10）
- 融合历史队列 `InstanceQueue`（`queue_length=4`）
- 通过 `temp_gnn + gnn + cross_gnn + ffn + refine`
- refine 同时输出：
  - actor motion 多模态轨迹
  - ego planning 多模态轨迹 + status

规划解码中有分层选择 + 可选 collision-aware rescore（配置 `use_rescore=True`）。

## 8) 关键输出字段释义（补充）

### 8.1 `map_head` 的 `vectors` 与 `scores`
- `vectors`：每个地图实例的折线点序列（polyline），由 `map_head` 最后一层回归输出 reshape 得到。
  - 默认每条向量形状为 `[20, 2]`（`num_sample=20`，2维平面坐标）。
  - 对应 `map_head` 回归维度 `40=20*2`。
- `scores`：对应每条 `vector` 的分类置信度（sigmoid 后分数，经过 top-k 筛选）。
- `labels`：类别 id（`ped_crossing/divider/boundary`）。

### 8.2 运动预测中的 `anchor_queue` 与 `period`
- `anchor_queue`：每个目标在历史队列中的锚框序列（历史时刻的状态锚）。
  - 来自 `InstanceQueue` 的缓存，推理返回前会 `torch.stack(..., dim=2)` 拼成时间维。
  - 每个锚是 3D 状态向量（位置、尺寸、朝向、速度等，和 det anchor 同编码空间）。
- `period`：该目标当前有效历史长度（可理解为“连续跟踪到的帧数”）。
  - 每个时间步递增，并裁剪到 `queue_length`。
  - 新出现/匹配中断的目标会有更短 `period`。

### 8.3 规划中的 `planning` 与 `final_planning`
- `planning`：候选轨迹集合，而不是单条轨迹。
  - 形状为 `[3, ego_fut_mode, ego_fut_ts, 2]`。
  - 其中 `3` 对应 command 组（left/straight/right 风格分组），`ego_fut_mode` 是每组候选模态数。
- `final_planning`：从 `planning` 中选出的最终一条轨迹。
  - 形状为 `[ego_fut_ts, 2]`。
- `planning_score`：候选轨迹对应分数；选择时可叠加 collision-aware rescore（配置 `use_rescore=True`）。

## 9) Stage1 / Stage2 训练策略

- `stage1`：只训练感知任务（`with_det=True, with_map=True, with_motion_plan=False`）。
- `stage2`：加载 `stage1` 权重后，训练 det+map+motion+planning 联合任务（`with_motion_plan=True`）。
- `stage2` 不是“冻结 backbone + 感知只训规划”：
  - backbone 配置是 `frozen_stages=-1`，并未冻结；
  - 只是通过 `paramwise_cfg` 给 `img_backbone` 更小学习率（`lr_mult=0.1`）做稳态微调。
